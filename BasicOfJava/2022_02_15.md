# 1. 상속(Inheritance)

## 1.1 상속의 의미  
&nbsp;상속이란 현실 세계에서 부모님이 자식들에게 재산을 물려주는 것과 유사합니다. 자식 클래스(서브 클래스)가 상속받고 싶은 부모 클래스(슈퍼 클래스)를 선택해서 물려받습니다.  
 - 현실 세계 : 경제적 + 사회적
 - 자바 세계 : 자산적(필드, 메서드) + 사회적(데이터 타입 동일시)

## 1.2 상속은 왜 쓰이는가?
 - 중복된 코드를 줄일 수 있습니다.
 - 유지 보수가 편리합니다.
 - 공통된 부분에 대한 클래스에 추가하여 확장성있게 사용이 가능합니다.
 - 개발 시간을 줄여줍니다.

## 1.3 상속하는 방법
```java
public class Parent {..} //부모 클래스
public class Child extends Parent {..} //자식 클래스

public class Myservlet extends HttpServlet
public class MyThread extends Thread
```
 - 이와 같이 extends 키워드를 써서 class A extends B 형태로 만들어줄 수 있습니다.
 - A : 자식 클래스 [추가할 부분] / B : 부모 클래스 [공통된 부분]

## 1.4 상속시 주의사항
 - 부모 클래스의 private 접근 제한을 갖는 필드 또는 메서드는 자식이 물려받을 수 없습니다.
 - 부모 클래스와 자식 클래스가 서로 다른 패키지에 있다면, 부모의 default 접근 제한을 갖는 필드 또는 메서드도 자식이 물려받을 수 없습니다.
 - 생성되는 과정을 보면 자식 객체가 생성되고 부모 객체가 생성되는 것으로 보이지만 사실 생성자를 통해 부모 객체가 먼저 생성되고 그 후, 자식 객체가 생성됩니다. 

## 1.5 객체 생성시 부모 생성자 관계
 - 상속시 기본적으로 부모 클래스의 디폴트 생성자를 호출하도록 되어 있습니다.
 - 필요시 부모 클래스의 디폴트 생성자가 아닌 파라미터가 있는 생성자를 호출해야할 때가 있습니다. 이 때 super(parameter) 메서드를 이용하면 디폴트 생성자를 호출하지 않고 부모 객체를 생성할 수 있습니다.
